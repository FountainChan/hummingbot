# Hummingbot 安装与启动指南

## 概述

Hummingbot 是一个开源的自动化交易框架，可以帮助你在多个交易所上设计和部署交易策略。本文档介绍如何快速地安装并启动 Hummingbot。

## 系统需求

- **操作系统**：Linux（推荐）、macOS 或 Windows
- **Python**：3.8+
- **Docker**：用于容器化部署（推荐）
- **内存**：最少 2GB
- **网络**：稳定的网络连接，能够访问对应的交易所 API

## 安装方式

### 方法一：使用 Docker（推荐）

#### 前置条件
- 安装 Docker 和 Docker Compose

#### 安装步骤

```bash
# 1. 克隆仓库
git clone https://github.com/hummingbot/hummingbot.git
cd hummingbot

# 2. 运行设置和部署
make setup
make deploy

# 3. 进入容器
docker attach hummingbot
```

### 方法二：从源代码安装

#### 前置条件
- Python 3.8+
- pip 和 conda（推荐）
- Build tools（gcc、g++、make等）

#### 安装步骤

```bash
# 1. 创建 Python 虚拟环境
conda create -n hummingbot python=3.10
conda activate hummingbot

# 2. 克隆仓库
git clone https://github.com/hummingbot/hummingbot.git
cd hummingbot

# 3. 安装依赖
pip install -e .

# 4. 启动 Hummingbot
hummingbot
```

## 首次启动

首次启动 Hummingbot 时，系统会引导你进行以下配置：

1. **选择策略**：从可用的策略列表中选择
2. **配置交易对**：选择要交易的交易对
3. **设置交易所**：连接到你的交易所账户（需要 API 密钥）
4. **参数配置**：根据你的策略调整参数

## 常用命令

```bash
# 启动 Hummingbot
hummingbot

# 查看帮助
hummingbot -h

# 使用特定配置文件启动
hummingbot -c /path/to/config.yml
```

## Docker 常用操作

```bash
# 查看运行中的容器
docker ps

# 进入容器交互模式
docker attach hummingbot

# 查看容器日志
docker logs hummingbot

# 停止容器
docker stop hummingbot

# 删除容器
docker rm hummingbot
```

## 配置文件位置

在 Docker 中运行时：
```
/root/hummingbot/conf/  # 配置文件目录
/root/hummingbot/logs/  # 日志文件目录
```

在本地运行时：
```
~/.hummingbot/conf/     # 配置文件目录
~/.hummingbot/logs/     # 日志文件目录
```

## 交易所 API 密钥配置

1. 在你的交易所账户中生成 API 密钥和密钥
2. 启动 Hummingbot 时，在提示时输入
3. Hummingbot 会加密存储你的密钥

## 故障排除

### 问题：无法连接到交易所
- 检查网络连接
- 验证 API 密钥是否正确
- 检查交易所 API 是否可用

### 问题：策略启动失败
- 检查配置文件语法
- 验证交易对信息是否正确
- 查看日志文件了解具体错误信息

### 问题：Docker 启动失败
- 确保 Docker Daemon 正在运行
- 检查系统资源是否充足
- 查看 Docker 日志获取更多信息
