## 🔍 按功能分类

### 交易策略

#### 方向性交易
- **技术指标** → 使用 Bollinger Bands、SuperTrend、MACD、RSI 等（[查看](核心概念/架构设计.md)）
- **参考代码** → `controllers/directional_trading/`
- **执行器** → OrderExecutor 或 PositionExecutor（[详见](执行器/执行器系统.md)）

#### 做市商
- **简单做市** → PMM Simple Controller（[详见](控制器/控制器系统.md#6-pmm-simplecontroller简单做市)）
- **动态做市** → PMM Dynamic Controller（[详见](控制器/控制器系统.md#7-pmm-dynamiccontroller动态做市)）
- **执行器** → GridExecutor（[详见](执行器/执行器系统.md#3-gridexecutor网格交易器)）

#### 套利
- **跨交所** → XemmExecutor（[详见](执行器/执行器系统.md#6-xemmexecutor跨交所做市执行器)）

- **参考代码** → `scripts/simple_xemm_gateway.py`

#### 分批执行
- **时间加权** → TWAPExecutor（[详见](执行器/执行器系统.md#4-twapexecutor时间加权平均价格执行器)）

## 💻 常用命令速查

### Docker 命令

```bash
# 查看容器状态
docker ps -a

# 查看日志（实时）
docker logs -f hummingbot

# 进入容器
docker exec -it hummingbot bash

# 停止/启动容器
docker stop hummingbot
docker start hummingbot

# 查看容器资源使用
docker stats hummingbot
```

### Hummingbot 命令

```bash
# 安装
pip install -e .

# 启动
hummingbot

# 使用指定配置
hummingbot -c conf/my_strategy.yml

# 查看版本
hummingbot -v

# 回测
hummingbot -c conf/my_strategy.yml --backtest
```

### 开发命令

```bash
# 运行测试
pytest test/

# 代码格式化
black hummingbot/

# 代码检查
flake8 hummingbot/

isort hummingbot/
```

## 📊 性能指标速查

### 关键 KPIs

```python
# 利润因子（越大越好）
Profit Factor = 总利润 / 总亏损

# 夏普比率（越高越好）
Sharpe Ratio = (平均收益 - 无风险收益率) / 标准差

# 最大回撤（越小越好）
Max Drawdown = (峰值 - 谷值) / 峰值 * 100%

# 胜率（越高越好）
Win Rate = 盈利交易数 / 总交易数 * 100%

# 收益率（越高越好）
Return = 总利润 / 初始资金 * 100%
```

## 🐛 故障排查速查表

| 现象 | 可能原因 | 解决方案 |
|------|---------|--------|
| 无法连接交易所 | API 密钥错误、网络问题 | 验证密钥、检查网络、查看日志 |
| 订单无法执行 | 余额不足、交易对错误、流动性不足 | 检查账户状态、验证交易对格式 |
| 内存占用过高 | K 线数据过多、日志堆积 | 减少 max_records、清理日志 |
| API 限流错误 | 请求过于频繁 | 增加 update_interval、批量请求 |
| 控制器不工作 | K 线数据不足、配置错误 | 增加 max_records、检查配置 |
| 策略经常崩溃 | 异常未处理、资源不足 | 查看详细日志、增加系统资源 |

## 🤔 常见问题

**Q: 我应该从哪里开始？**
A: 从 [快速开始](快速开始/安装与启动.md) 和 [架构设计](核心概念/架构设计.md) 开始。

**Q: 如何选择 ScriptStrategyBase 还是 StrategyV2？**
A: 简单策略用 ScriptStrategyBase，复杂多算法用 StrategyV2。[详见](策略开发/新策略开发指南.md#方式一使用-scriptstrategeybase简单策略)

**Q: 执行器和控制器有什么区别？**
A: 控制器生成信号，执行器执行订单。[详见](核心概念/架构设计.md#控制器系统)

**Q: 如何在生产环境运行？**
A: 使用 Docker 和云服务器。[详见](部署运维/部署指南.md)

**Q: 遇到问题怎么办？**
A: 查看 [故障排查](部署运维/部署指南.md#4-故障排查) 部分，或访问 [Discord 社区](https://discord.gg/hummingbot)。
